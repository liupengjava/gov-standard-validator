# 群众感知比对三阶段功能 PRD

日期：2026-07-23

## 1. 功能定位

在标准验证智能体中增加“群众感知比对”能力，用公开舆情、热线留言、问卷访谈、基层调研等样本，辅助判断标准条款是否贴近群众真实办事体验。

本功能不改变当前报告主依据。正式验证结论仍以“待验证文本与现有标准知识库、法律法规、国家标准、行业标准、地方细则”的条款级比对为核心；群众感知只作为辅助佐证，用于补充问题现实性、影响范围、整改优先级和专家采纳建议。

## 2. 背景与问题

当前平台已经具备标准知识库切片、待验证文本切片、条款召回、规则校验、舆情与调研样本管理、格式化报告输出等基础能力。现有智能体执行过程主要展示“读取输入、结构抽取、规则校验、知识库比对、结论生成”，但舆情样本尚未明确进入验证链路，也没有在报告中形成稳定的证据表达。

需要将舆情能力按阶段纳入产品，而不是一次性改变主验证逻辑，避免报告结论被非标准来源稀释。

## 3. 建设目标

1. 在智能体执行窗口中显式展示“舆情感知比对”步骤，让用户知道系统是否完成群众感知分析。
2. 在报告中新增“群众感知佐证”模块，但标注为辅助依据，不覆盖标准知识库比对结论。
3. 后续支持将舆情与调研样本向量化，按条款、事项、问题标签进行召回，提升辅助证据匹配准确度。

## 4. 非目标

1. 不取消待验证文本切片确认流程。
2. 不把舆情样本作为合法性、合规性或标准一致性的主判断依据。
3. 不自动采信网络舆情为事实结论，必须保留来源、时间、地域、样本类型和可信度提示。
4. 本评审稿不进入开发实现，待确认后再拆分开发任务。

## 5. 三阶段范围

### 阶段一：执行过程纳入舆情感知比对

将当前 5 步执行过程扩展为 6 步：

1. 读取输入
2. 结构抽取
3. 规则校验
4. 知识库比对
5. 舆情感知比对
6. 结论生成

阶段一只要求前端执行窗口展示该步骤，并在验证日志中说明系统完成了样本检索、问题标签归类、条款关联和辅助证据汇总。

### 阶段二：报告增加群众感知佐证

在格式化验证报告中增加独立章节：

“群众感知佐证（辅助依据）”

建议内容结构：

1. 关联条款：对应本次验证的标准条款或待验证文本切片。
2. 感知摘要：群众反馈集中在哪些办事环节、材料要求、线上线下一致性、补正告知、进度查询、服务体验等问题。
3. 样本来源：公开留言、热线、问卷、访谈、基层调研、历史投诉等。
4. 影响判断：用于支持“问题是否真实存在”“是否影响群众权益”“是否应提高整改优先级”。
5. 证据边界：明确该部分为辅助佐证，不能替代标准、法规和专家审定。

### 阶段三：舆情样本向量化召回

将舆情与调研样本按统一结构入库并支持向量召回：

1. 样本切片：按事项、地域、来源、时间、问题对象、群众诉求拆分。
2. 向量入库：与标准知识库向量区区分存储，但在验证时可联合召回。
3. 条款关联：通过关键词、事项标签、语义相似度和问题标签关联到待验证条款。
4. 召回展示：在报告和执行过程明细中展示匹配样本、相似度、来源可信度和证据链。

## 6. 数据输入

1. 标准知识库条款：现有切片入库后的知识库向量。
2. 待验证文本：保留当前上传、解析、切片、确认流程。
3. 舆情与调研样本：现有手工导入、批量解析、接口采集和公开来源采集。
4. 专家确认结果：逐项关键验证点的采纳或拒绝状态。

## 7. 数据输出

1. 智能体执行步骤：新增“舆情感知比对”步骤和进度状态。
2. 验证点辅助信息：每个关键验证点可显示关联群众反馈数量和典型摘要。
3. 报告章节：新增“群众感知佐证（辅助依据）”。
4. 证据清单：标准证据与群众感知证据分层展示。

## 8. 报告主次规则

报告结论排序建议固定为：

1. 主依据：标准知识库、法律法规、国家标准、行业标准、地方细则、办事指南。
2. 核心判断：一致、部分一致、冲突、缺失、表述偏弱、需专家复核。
3. 辅助佐证：群众感知、舆情样本、问卷访谈、基层调研。
4. 专家意见：最终采纳、拒绝、补充说明。

系统文案必须避免把群众反馈写成“证明条款错误”。推荐表述为“提示该条款存在现实执行关注点”“支持提高复核优先级”“作为专家研判参考”。

## 9. UI 设计要求

1. 执行过程区域：在“知识库比对”之后增加“舆情感知比对”，用辅助色和“辅助依据”标签区分主流程。
2. 当前步骤明细：展示召回样本数、关联条款数、典型问题标签、证据边界提示。
3. 报告预览区域：先展示“标准知识库比对主结论”，再展示“群众感知佐证”，保持主次。
4. 不在正式页面展示顶部“三阶段说明/群众感知比对进入验证链路”区域，三阶段只保留在 PRD 和内部建设规划中。

## 10. 验收标准

1. 智能体执行过程显示 6 步，且“舆情感知比对”位于“知识库比对”和“结论生成”之间。
2. 在没有舆情样本时，系统仍可完成验证报告，但“群众感知佐证”显示为空状态或“暂无可用辅助样本”。
3. 在存在舆情样本时，报告能展示关联摘要、来源类型、样本数量和证据边界。
4. 报告主结论仍来自标准知识库比对，不因舆情样本数量多而自动改变合规判断。
5. 舆情样本入库和召回结果支持分页查看，后续与现有向量区设计保持一致。

## 11. 风险与处理

1. 舆情来源真实性不足：保留来源、发布时间、采集方式和可信度，不自动认定事实。
2. 样本与条款误匹配：阶段三加入相似度阈值、关键词命中和人工复核。
3. 报告主次混淆：UI 和报告文案持续标注“辅助依据”。
4. 数据合规风险：对涉及个人信息的样本进行脱敏，避免展示身份证号、手机号、姓名等敏感信息。
5. 网络采集不稳定：允许手工导入和离线样本作为兜底。

## 12. 已确认事项

1. 阶段一直接接入现有舆情与调研样本统计，不只做静态模拟。
2. 报告中“群众感知佐证”章节放在“标准知识库比对主结论”之后，作为辅助依据展示。
3. 阶段三舆情样本向量区放在“舆情与调研”页面单独展示，与标准知识库向量区在信息结构上并列但不混放。
